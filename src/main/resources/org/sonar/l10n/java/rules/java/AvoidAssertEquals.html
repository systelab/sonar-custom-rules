Using assertEquals from JUnit is discouraged in favor of AssertJ's fluent assertions. AssertJ provides a more readable and maintainable assertion API, improving test clarity and debugging information.

Noncompliant Code Example

class MyClass {
  private String expectedValue = "some expected value";
  void checkSomething(String a) {
     assertEquals(expectedValue, a); // Noncompliant
  }
}

Compliant Solution

class MyClass {
  private String expectedValue = "some expected value";
  void checkSomething(String a) {
     assertThat(actualValue).isEqualTo(expectedValue);  // Compliant
  }
}
